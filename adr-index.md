# ADR Index — Exeris Ecosystem

**Single numbering namespace** across all Exeris repositories. ADR files live next to the code they describe; this index is the canonical lookup (number → owning repo + link).

## Rules

1. **Tech-side ADRs only.** Business decisions live in `business-adr-index.md` (`BUS-NNN` namespace). Refactoring-only ADRs do not enter the registry.
2. **Number reservation.** A number is reserved by a PR to this file before the ADR-content PR merges.
3. **Visibility.** Per ADR-020.
   - `public` — content lives in a public open-core repo.
   - `enterprise-private` — content lives only in a private enterprise repo. Number is publicly registered here, content is not.
4. **Scope.**
   - `per-repo` — bounded to one repo.
   - `cross-repo` — affects multiple repos. Owning repo holds the authoritative copy; every other affected repo has a `docs/adr/ADR-NNN.link.md` stub.
   - `platform` — stack-level recommendation. Owning repo: `exeris-docs`.
5. **Routines and reviewers cite by number**, not by path.
